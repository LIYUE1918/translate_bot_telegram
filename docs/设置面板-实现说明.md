# 设置面板（Inline Keyboard）实现说明

本文档介绍项目中“设置面板”选项是如何实现的：使用 Telegram Bot 的内联按钮（InlineKeyboard）展示并修改机器人配置，包括源语言、目标语言、翻译引擎与计时显示。

## 功能概览

- 通过命令 `/settings` 弹出设置面板。
- 面板包含多个按钮：
  - 源语言：auto / en / zh
  - 目标语言：en / zh
  - 引擎：bing / google
  - 切换计时：开启或关闭耗时显示
- 点击按钮后立即生效，并在同一条消息内“原地刷新”当前设置。

## 关键依赖

- Telegram 对象与处理器：
  - `InlineKeyboardButton`, `InlineKeyboardMarkup`（构建按钮与面板）
  - `CallbackQueryHandler`（处理按钮回调）
- 参考实现位置：[main.py](file:///c:/Users/QWQ/Desktop/%E6%9C%BA%E5%99%A8%E4%BA%BA/v4/main.py)

## 数据结构与默认值

- 使用 `context.user_data` 为每位用户单独保存设置：
  - `translate_src`：源语言（默认 `auto`）
  - `translate_tgt`：目标语言（默认 `zh`）
  - `translate_mode`：引擎（默认 `bing`）
  - `translate_count`：是否显示耗时（默认 `True`）
- 默认值常量定义在 [main.py](file:///c:/Users/QWQ/Desktop/%E6%9C%BA%E5%99%A8%E4%BA%BA/v4/main.py#L9-L13) 顶部：`DEFAULT_SRC`、`DEFAULT_TGT`、`DEFAULT_MODE`、`DEFAULT_COUNT`。

## 展示面板（/settings）

入口命令将当前设置格式化为文本，并附上内联按钮：

```python
async def settings(update: Update, context: ContextTypes.DEFAULT_TYPE):
    if "translate_tgt" not in context.user_data and "translate_lang" in context.user_data:
        context.user_data["translate_tgt"] = context.user_data.get("translate_lang", DEFAULT_TGT)
    text = _settings_text(context)
    keyboard = [
        [InlineKeyboardButton("源: auto", callback_data="set_src:auto"),
         InlineKeyboardButton("源: en",   callback_data="set_src:en"),
         InlineKeyboardButton("源: zh",   callback_data="set_src:zh")],
        [InlineKeyboardButton("目标: en",  callback_data="set_tgt:en"),
         InlineKeyboardButton("目标: zh",  callback_data="set_tgt:zh")],
        [InlineKeyboardButton("引擎: bing",   callback_data="set_engine:bing"),
         InlineKeyboardButton("引擎: google", callback_data="set_engine:google")],
        [InlineKeyboardButton("切换计时", callback_data="toggle_count")],
    ]
    await update.message.reply_text(text, reply_markup=InlineKeyboardMarkup(keyboard))
```

## 处理按钮回调

按钮点击后会携带 `callback_data`，由 `CallbackQueryHandler` 分发至回调函数，在其中解析并更新 `user_data`，然后用 `edit_message_text` 原地刷新：

```python
async def on_settings_callback(update: Update, context: ContextTypes.DEFAULT_TYPE):
    query = update.callback_query
    data = (query.data or "")
    if data.startswith("set_src:"):
        context.user_data["translate_src"] = data.split(":", 1)[1]
    elif data.startswith("set_tgt:"):
        context.user_data["translate_tgt"] = data.split(":", 1)[1]
    elif data.startswith("set_engine:"):
        context.user_data["translate_mode"] = data.split(":", 1)[1]
    elif data.startswith("toggle_count"):
        cur = context.user_data.get("translate_count", DEFAULT_COUNT)
        context.user_data["translate_count"] = not cur
    await query.answer()
    keyboard = [...同上...]
    await query.edit_message_text(_settings_text(context), reply_markup=InlineKeyboardMarkup(keyboard))
```

## 翻译逻辑如何读取设置

消息到达 `echo` 时，从 `user_data` 读取源/目标/引擎/计时进行翻译与计时：

```python
async def echo(update: Update, context: ContextTypes.DEFAULT_TYPE):
    text = update.message.text or ""
    src = context.user_data.get("translate_src", DEFAULT_SRC)
    tgt = context.user_data.get("translate_tgt", DEFAULT_TGT)
    engine = context.user_data.get("translate_mode", DEFAULT_MODE)
    show_time = context.user_data.get("translate_count", DEFAULT_COUNT)
    start = time.perf_counter()
    if engine == "google":
        res = ts.translate_text(text, from_language=src, to_language=tgt)
    elif engine == "bing":
        res = ts.translate_text(text, from_language=src, to_language=tgt, translator="bing")
    else:
        res = ts.translate_text(text, from_language=src, to_language=tgt, translator=engine)
    elapsed = time.perf_counter() - start
    msg = res if not show_time else f"{res}\n\n[{engine} {elapsed:.3f}s]"
    await update.message.reply_text(msg)
```

## 常见扩展

- 添加更多语言：在 `/settings` 的键盘里新增按钮并在回调里处理 `set_src:xx`/`set_tgt:xx`。
- 添加更多引擎：扩展引擎按钮与回调；在 `echo` 中增加分支或统一用 `translator=engine`。
- 持久化设置：将 `user_data` 落地，可使用数据库或本地存储（需自行实现）。
- 权限与失败：`edit_message_text` 需要机器人可编辑该条消息；如失败，可回退为发送新消息。

## 小结

设置面板的核心在于：
1. 用 `InlineKeyboardMarkup` 构建按钮；
2. 用 `CallbackQueryHandler` 解析 `callback_data` 并更新 `user_data`；
3. 用 `edit_message_text` 原地刷新显示；
4. 业务侧（echo）只关心从 `user_data` 读取最新配置执行翻译。
